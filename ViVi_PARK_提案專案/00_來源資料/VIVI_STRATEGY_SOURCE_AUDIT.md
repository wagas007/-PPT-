## 2. 主要發現

### 發現一：原 Strategy 方向正確，但旺來策略權重不足

原文件已正確辨識 ViVi PARK 是「停車 + Vi 幣 + APP + 生活服務平台」，但較偏 ViVi 視角。  
若不加入旺來策略，後續文件容易變成：

```text
旺來想接入 ViVi 生活服務平台
```

但更精準的策略應該是：

```text
旺來希望以 ViVi PARK 作為 OMO 家庭能源服務的外部驗證通路，
同時測試會員導流、點數 / 商品券、線下智能自取節點三種合作可能。
```

### 發現二：ViVi 與旺來不是單純衝突，而是平台相似、品類互補

相似處：

- 都在做線上會員平台。
- 都需要線下服務節點。
- 都重視金流、點數、票券或會員轉換。
- 都希望把單一服務擴展為生活服務生態圈。

差異處：

- ViVi 的底層場景是停車、車主、加油、充電與生活消費。
- 旺來的底層場景是 LPG 家庭能源、瓦斯回購、首配安檢、智能自取與配送。

結論：  
不宜把 ViVi 定義成競爭者；應定義為「可能共同驗證家庭能源生活服務的停車會員平台」。

### 發現三：金流與零售通路資訊會改變合作切入

公開資料顯示：

- ViVi PARK 會員條款列出信用卡刷卡、ATM 匯款或購買頁顯示之付款方式。
- ViVi PARK 官網最新消息列出「新增付款方式 - 富邦金流」。
- ViVi PARK 已上架 LINE Pay 購好券，可在 LINE Pay App 購買 Vi 幣點數。
- ViVi PARK 官方關於我們頁面揭露，2020 年獲全聯委託經營停車場。
- ViVi PARK 場站資料中有多個全聯場站。

因此後續提案不能只談場站設置，也要問：

```text
旺來服務能否以商品券 / 服務券 / Vi 幣消費場景上架？
旺來首購或會員招募能否透過 ViVi 會員導流？
智能自取櫃是否只是其中一個履約節點，而非唯一合作方式？
```

## 4. 建議後續重做方向

### OnePager

舊主軸：

```text
停車後生活服務節點
```

建議新主軸：

```text
ViVi PARK × 旺來瓦斯
家庭能源生活服務節點試點合作
```

OnePager 應呈現三種合作可能：

1. 會員導流：ViVi 車主會員導入旺來首購 / 家庭能源服務。
2. 點數 / 商品券：Vi 幣、LINE Pay 購好券、服務券合作可能。
3. 智能自取節點：桃園 / 台中 1 至 3 場低干擾試點。

### Coldmail

信件不應一開始只談場站。建議開場改為：

```text
我們留意到 ViVi PARK 正在拓展停車、Vi 幣、點數漫遊與生活服務生態圈。
旺來希望交流一個家庭能源服務接入 ViVi 會員與場站場景的合作構想。
```

### PPT

若後續製作簡報，頁序應先講：

```text
旺來 2026 策略
→ ViVi 的平台能力
→ 雙方相似但互補
→ 三條合作路徑
→ 先選最適合的小規模驗證
```

不要一開始直接進入場站清單。

## 5. 待確認問題

下一次與 ViVi 交流時，應優先確認：

1. 富邦金流是否只支援 Vi 幣儲值，或也可承接第三方商品券 / 服務券？
2. LINE Pay 購好券合作是 Vi 幣銷售通路，還是可擴展到其他服務商品？
3. 全聯場站的停車場權責由 ViVi、全聯或地主哪一方決定？
4. 生活服務 / 特約商店上架是否可接受 LPG / 家庭能源服務？
5. 若不先設櫃，是否可先做會員導流或首購券測試？
6. 若做場站試點，桃園 / 台中哪些場站權責單純、供電可確認？

## 6. 總結

ViVi PARK 專案不應被降格成「停車場放置設備」。  
更好的策略是把它定義為：

```text
旺來 OMO 家庭能源服務，接入 ViVi PARK 停車會員、點數票券與線下場站節點的合作驗證。
```

因此，Strategy / Execution / Report 已完成第一輪重估；後續 OnePager、Coldmail、PPT 應依此重新設計。
