# 嘟嘟房 BD 自動化工作流 Prompt

> 使用目的：本檔用來指揮 Codex 依序產出嘟嘟房提案專案的 BD 開發文件。  
> 重要規則：每個節點完成後都必須停下來，等待使用者確認，不可自動進入下一個節點。

## 使用方式

每次只複製一個節點的 Prompt 給 Codex。

若使用語音輸入，可以直接說：

```text
請依照 嘟嘟房_提案專案/99_工作暫存/DODO_BD_AUTOMATION_PROMPTS.md 的第 X 節執行。
完成後請停下來，不要進入下一步。
```

## 全流程節點

```text
-1. DD 調查建立與來源資料產出          ← 必須最先執行
-0.5. 案場資料庫建立（桃園 / 台中）   ← DD 完成後、盤點前執行
 0. 專案資料盤點
 1. 嘟嘟房專案定位分析
 2. OnePager 內容稿
 3. OnePager DM HTML
 4. PNG / PDF 輸出
 5. Coldmail 工作流與寄送文件
 6. 追蹤話術與取得會議文件
 7. 簡版 PPT 大綱
 8. 正式 PPT 大綱或 HTML 預覽
```

重要：
- 節點 -1 是所有節點的前提。
- 節點 -0.5（案場資料庫）在節點 -1 之後、節點 0 之前執行。
- 若 00_來源資料 尚未有 DD 調查產出的文件，請先執行節點 -1，才能進入節點 -0.5。
- 若 DODO_案場資料庫_v1.xlsx 尚未建立，請先執行節點 -0.5，才能進入節點 0。

每個節點都要遵守：

```text
只完成本節指定產出。
完成後回報產出檔案。
不要自動執行下一節。
等待使用者確認後再繼續。
```

## -0.5. 案場資料庫建立（桃園 / 台中）Prompt

```text
請先閱讀：
- 嘟嘟房_提案專案/00_來源資料/DODO_DD_REPORT.md（場站規模、城市分布）
- 嘟嘟房_提案專案/00_來源資料/DODO_PROJECT_MASTER.md（場站清單框架）
- PARK24_Times_提案專案/00_來源資料/PARK24_Project_Master.xlsx（參考格式）
- 城市車旅_提案專案/00_來源資料/CITY_案場資料庫_v1.xlsx（參考格式）

這是嘟嘟房 BD 流程的第 -0.5 步，在 DD 調查完成後、進入資料盤點（Step 0）之前執行。

目標：建立嘟嘟房在桃園市與台中市的候選場站資料庫，作為 OnePager HTML、Cold mail 與試點選址的資料基礎。

請執行以下任務：

1. 搜尋嘟嘟房官方網站場站查詢頁面（https://www.dodohome.com.tw），
   收集桃園市與台中市的場站清單，包含：
   - 場站名稱
   - 地址（含區域）
   - 場站型態（地下/平面/立體）
   - 支付方式（可推測供電條件）

2. 按照 CITY_案場資料庫_v1 的評分模型（合作價值 1-5、導入難度 1/3/5、優先級 A/B/C），
   對每個場站進行評分。

3. 按照 PARK24_Project_Master 的供電分析邏輯（Availability / Accessibility / Economics），
   對 TOP10 場站進行供電可行性評估。

4. 更新 DODO_案場資料庫_v1.xlsx 的以下工作表：
   - 03_台中場站庫（補入實際場站地址與評分）
   - 04_桃園場站庫（補入實際場站地址與評分）
   - 06_TOP10初選（依評分更新推薦排序）
   - 07_供電分析（依 TOP10 補入供電評估）

   若無法取得完整官方場站清單，請：
   - 保留現有框架結構
   - 在備註欄標記「⚠️待嘟嘟房提供官方場站清單後更新」
   - 盡可能補入可從公開資料確認的場站資訊

要求：
1. 使用繁體中文。
2. 桃園與台中各需至少 5 個評分完成的候選場站（可標待驗證）。
3. TOP10 至少包含桃園 3 站、台中 3 站、其他城市 4 站（備選）。
4. 供電分析需明確標出下一步行動（請嘟嘟房提供電箱位置或現勘確認）。
5. 不要製作 OnePager、HTML 或 PPT。
6. 完成後回報更新了哪些工作表，然後停下來，等待確認，再進入節點 0。
```
